## About this roadmap

This repository contains a personal cybersecurity learning roadmap that I originally designed to recreate, independently and manually, the progression of a local university cybersecurity degree.

It is not an official curriculum, a complete course, or a promise that every listed topic must be studied in the exact order shown. It is a structured reference for building broad foundations first, progressing into offensive and defensive security, and eventually choosing deeper specialization areas.

The roadmap combines:

- Computer science, mathematics, systems, and networking foundations.
- Offensive security, defensive security, governance, and incident response.
- Practical laboratories, CTFs, bug bounty platforms, and public references.
- Optional certifications aligned with specific skills.
- Projects and evidence of learning instead of passive content consumption.
- English study throughout the entire path.

> [!NOTE]
> This is a living roadmap. Resources, certifications, and priorities may change as technologies and my own learning goals evolve.

## How to use it

Do not treat the roadmap as a checklist that must be completed mechanically.

For each subject:

1. Learn the underlying concepts.
2. Practice them in a controlled environment.
3. Build or analyze something that uses them.
4. Document what worked, what failed, and why.
5. Keep evidence that the skill can be applied again.

A subject is not complete because a video, room, or book was finished. It is complete when the knowledge can be explained and applied without following instructions blindly.

## Curriculum

The curriculum is divided into three progressive training cycles and ten terms. The sixth and tenth terms are reserved for substantial practical work or internships.

### First training cycle

#### Term 1 — Foundations

- [ ] Mathematical Foundations
- [ ] Cybersecurity Fundamentals
- [ ] Introduction to Operating Systems and Networks
- [ ] Operating System Security
- [ ] Communication and Digital Skills
- [ ] English I
- [ ] Mathematics I

**Suggested evidence**

- Configure and document a small virtualized laboratory.
- Explain the main components of an operating system and a TCP/IP network.
- Produce a basic threat model for a personal system or small application.
- Complete introductory Linux, Windows, networking, and security laboratories.

#### Term 2 — Networking and Initial Offensive Security

- [ ] Introduction to Penetration Testing
- [ ] Exploitation and Post-Exploitation
- [ ] Probability and Statistics
- [ ] Network Security
- [ ] Cybersecurity Governance
- [ ] English II
- [ ] Mathematics II

**Suggested evidence**

- Perform a scoped penetration test against a deliberately vulnerable laboratory.
- Document reconnaissance, enumeration, exploitation, and remediation.
- Analyze basic network behavior with Wireshark or tcpdump.
- Create a small dataset from security experiments and summarize its results.

#### Term 3 — Traffic, Identity, Data, and Risk

- [ ] Network Traffic Analysis
- [ ] Technology Risk Management
- [ ] Identity and Access Control
- [ ] Data Protection
- [ ] Integrative Project I
- [ ] English III
- [ ] Mathematics III

**Suggested evidence**

- Analyze packet captures and identify protocols, anomalies, and attack indicators.
- Design a basic identity and access-control model.
- Create a risk register for a small system.
- Build an integrative project combining networking, systems, and security controls.

#### Term 4 — Secure Development and Offensive Methodology

- [ ] Social Engineering
- [ ] Critical Security Controls I
- [ ] Secure Development Foundations
- [ ] Hacking Foundations
- [ ] DevSecOps
- [ ] Intelligence and National Security Foundations
- [ ] English IV
- [ ] Mathematics IV

**Suggested evidence**

- Threat-model and harden a small application.
- Add automated security checks to a development pipeline.
- Create an authorized social-engineering assessment plan without targeting real users.
- Compare security controls against a documented attack path.

#### Term 5 — Vulnerability Management and Cyberdefense

- [ ] Physical Hacking
- [ ] Vulnerability Management
- [ ] Cyberdefense Foundations
- [ ] Critical Security Controls II
- [ ] Open-Source Intelligence Techniques
- [ ] Integrative Project II
- [ ] English V
- [ ] Mathematics V

**Suggested evidence**

- Build a vulnerability-management workflow from discovery to remediation verification.
- Conduct an OSINT investigation using only public and authorized data.
- Create a defensive monitoring laboratory.
- Complete an integrative project that produces reproducible evidence and a technical report.

#### Term 6 — Technical Internship or Major Project

- [ ] Technical internship or equivalent independent project
- [ ] Mathematics VI

The original progression places a technical internship here. An independent learner can replace it with a substantial project that integrates the first five terms.

Examples:

- A complete authorized penetration-testing engagement in a laboratory.
- A small security monitoring and incident-response environment.
- A vulnerability scanner, reconnaissance tool, or security automation project.
- A documented research experiment with public code and reproducible results.

### Second training cycle

#### Term 7 — Web Security, Architecture, and Threat Management

- [ ] Web Application Hacking
- [ ] Cyberdefense Architecture
- [ ] Threat Management
- [ ] Cryptography
- [ ] Compliance
- [ ] Hacking Techniques
- [ ] English VI
- [ ] Mathematics VII

**Suggested evidence**

- Complete a structured selection of PortSwigger Web Security Academy laboratories.
- Assess a deliberately vulnerable web application and produce a professional report.
- Design a security architecture for a small organization or cloud application.
- Implement and explain common cryptographic operations without inventing custom cryptography.

#### Term 8 — Mobile, Analytics, Incident Response, and Cloud

- [ ] Mobile Application Hacking
- [ ] Cybersecurity Analytics
- [ ] Incident Response and Management
- [ ] Digital Forensics
- [ ] Hardening
- [ ] Cloud Cybersecurity
- [ ] English VII
- [ ] Mathematics VIII

**Suggested evidence**

- Harden and audit a Windows or Linux host against a defined baseline.
- Investigate a simulated incident and preserve a clear evidence trail.
- Analyze logs or telemetry with scripts, queries, or notebooks.
- Deploy and secure a small cloud environment.

#### Term 9 — Advanced Analysis and Strategy

- [ ] Vulnerability Analysis and Hacking
- [ ] ICS, SCADA, and IoT Cybersecurity
- [ ] Malware Analysis
- [ ] Business Continuity and Disaster Recovery
- [ ] Cybersecurity Strategy
- [ ] Integrative Project III
- [ ] English VIII
- [ ] Mathematics IX

**Suggested evidence**

- Perform static and dynamic analysis of a harmless malware sample or educational binary.
- Build a basic continuity and disaster-recovery plan.
- Study an isolated industrial-control or IoT laboratory.
- Complete a final integrative project with methodology, results, limitations, and documentation.

#### Term 10 — Professional Internship or Capstone

- [ ] Professional internship or equivalent capstone project
- [ ] Mathematics X

A suitable independent capstone should solve a real problem and produce artifacts that another person can inspect, reproduce, or use.

Examples:

- An open-source cybersecurity tool.
- An original security research project.
- A multi-agent AI system for authorized security testing.
- A defensive platform, detection pipeline, or incident-response automation system.
- A documented security assessment of an application whose owner explicitly authorized the work.

## Evidence of learning

A useful learning path should produce observable evidence. Depending on the subject, evidence may include:

- Source code and automation scripts.
- Reproducible laboratory environments.
- Technical reports and remediation recommendations.
- Packet-capture or log analyses.
- Threat models and architecture diagrams.
- Security baselines and hardening documentation.
- Detection rules and incident-response playbooks.
- CTF or laboratory write-ups that respect platform rules.
- Research questions, hypotheses, datasets, experiments, and results.
- Contributions to open-source security projects.
- Certifications when they support a concrete professional objective.

## Suggested project progression

| Level | Project example |
| --- | --- |
| Foundation | Build and document a virtual cybersecurity laboratory |
| Beginner | Create a network or web reconnaissance utility |
| Intermediate | Perform and report an authorized assessment of a vulnerable application |
| Intermediate | Build a small log-analysis or vulnerability-management pipeline |
| Advanced | Develop an open-source security tool with tests and documentation |
| Advanced | Conduct a reproducible security research project with measurable results |
| Capstone | Integrate AI, automation, and security tools into a system that solves a real operational problem |

## Principles

- Build foundations before collecting advanced tools.
- Use laboratories to test knowledge, not replace it.
- Prefer reproducible evidence over claims of expertise.
- Learn both attack and defense to understand complete systems.
- Treat reports and documentation as part of the technical work.
- Use certifications strategically rather than collecting them without a purpose.
- Publish useful work after building it for a real reason, not merely to fill a profile.
- Update the roadmap when experience disproves an assumption.

## Origin and scope

This roadmap began as a personal attempt to manually reproduce the learning progression of a newly introduced cybersecurity degree at a local university. I reorganized the subjects into an independent path and supplemented them with practice platforms, references, certifications, specialization topics, and evidence-based outcomes.

The repository is not affiliated with or endorsed by that university or by any certification provider or platform linked above.
